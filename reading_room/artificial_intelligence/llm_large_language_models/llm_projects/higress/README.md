# Higress 项目简介

## 1. 项目定位

Higress 是一个基于 Envoy 与 Istio 生态的云原生网关项目，重点提供 API 网关、流量治理与插件扩展能力。

在 LLM 场景中，Higress 常用于统一模型服务入口，承接鉴权、限流、灰度、观测与安全策略，帮助将模型推理服务能力产品化、平台化。

## 2. 适合 LLM 平台的原因

- 网关能力完备：支持路由、负载均衡、熔断、重试等常见治理能力。
- 插件扩展友好：可通过插件快速叠加鉴权、审计、流控、内容安全等能力。
- 云原生兼容性好：适合 Kubernetes 环境，与 Service Mesh 体系可协同。
- 可观测性较强：便于监控请求时延、错误率、吞吐，支持问题定位。

## 3. 在推理平台中的典型位置

可将 Higress 放在“客户端/业务系统”与“模型推理引擎（如 vLLM、SGLang、TGI）”之间：

1. 对外暴露统一 API。
2. 将请求按模型、租户、版本路由到后端推理服务。
3. 通过策略实现限流、配额和多租户隔离。
4. 记录访问日志与指标，接入告警系统。

## 4. 常见能力清单

- API 路由与版本管理
- Token / QPS 限流
- 访问认证与鉴权（例如 API Key / JWT）
- 灰度发布与流量分配
- 审计日志与可观测
- 安全防护与异常请求拦截

## 5. 推荐调研方向

- Higress 与 Istio/Envoy 的关系及部署模型
- 插件机制与二次开发方式
- 在 LLM 网关中的性能与延迟开销
- 与现有推理框架（vLLM、SGLang 等）的接入实践
