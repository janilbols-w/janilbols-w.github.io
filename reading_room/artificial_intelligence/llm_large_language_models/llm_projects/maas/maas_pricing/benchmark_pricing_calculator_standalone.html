Benchmark To Pricing Calculator

输入同一套硬件下的 benchmark 数据，自动计算可售卖吞吐、每百万 token 成本、目标售价和最优生产并发。

怎么看这个页面

  • 左侧先填每小时成本，表示这套硬件与平台的固定小时支出。
  • 中间填定价参数与 SLA，表示你希望赚多少毛利，以及允许多大的延迟和错误率。
  • 下方表格逐行填写不同并发的 benchmark 结果，系统会自动算出价格与利润。

核心判断逻辑

  • `SLA PASS` 代表该并发点满足你设定的 TTFT、E2E 和错误率要求。
  • `Goodput` 是真正可售卖的请求吞吐，不是裸 benchmark 吞吐。
  • 高并发不一定最好，系统会优先找满足 SLA 且 `Cost / 1M` 最低的并发点；若成本相同再比较 `Goodput`。

价格结果怎么理解

  • `Cost / 1M` 是每百万 token 的真实成本。
  • `Price / 1M` 是按毛利率、支付费率、坏账、税费反推的基础售价。
  • `Final / 1M` 再叠加低延迟、可靠性、容量预留等溢价系数。

生成公式输入/策略定义

这部分是你设定的定价结构与业务假设，系统按这些公式做后续计算。

$$C_{hour}=C_{gpu}+C_{cpu}+C_{mem}+C_{storage}+C_{network}+C_{gateway}+C_{obs}+C_{ops}+C_{risk}$$
$$\hat{w}_{cache}=\frac{w_{cache}}{w_{cache}+w_{in}+w_{out}},\;\hat{w}_{in}=\frac{w_{in}}{w_{cache}+w_{in}+w_{out}},\;\hat{w}_{out}=\frac{w_{out}}{w_{cache}+w_{in}+w_{out}}$$
$$r_{net}=1-m-f_{pay}-f_{bad}-f_{tax}$$
$$P_{base/1M}=\frac{Cost_{1M}}{r_{net}},\quad P_{final/1M}=P_{base/1M}\times K_{lat}\times K_{rel}\times K_{reserve}$$
$$P_{in}=\frac{P_{final/1M}}{\alpha\hat{w}_{cache}+\hat{w}_{in}+\beta\hat{w}_{out}},\quad P_{cache}=\alpha P_{in},\quad P_{out}=\beta P_{in}$$

推导结果由 benchmark 数据计算

这部分是系统根据每个并发点数据推导出来的结果，用于选最优生产并发。

$$SLA\_PASS_c=(TTFT95_c\le T_{sla})\land(E2E95_c\le E_{sla})\land(ERR_c\le e_{sla})$$
$$S_c=\begin{cases}\text{用户填写的 SLA Ratio},&\text{若填写}\\1,&\text{若未填写且 }SLA\_PASS_c=\text{true}\\0,&\text{若未填写且 }SLA\_PASS_c=\text{false}\end{cases}$$
$$G_c=R_c\times S_c$$
$$T_c=3600\times G_c\times(I_c+O_c)$$
$$Cost_{1M,c}=\frac{C_{hour}}{T_c}\times10^6$$
$$Profit_{h,c}=\left(\frac{P_{final/1M,c}}{10^6}\right)\times T_c-C_{hour}$$
$$c^*=\arg\min_{c\in\{SLA\_PASS_c=true\}}Cost_{1M,c},\;\text{tie-break by }G_c\text{ then lower }c$$

参数定义总览（Standalone 完整版）

一、成本与定价参数（输入）

  • C_hour：每小时总成本（单位：CN¥/h），C_gpu + C_cpu + C_mem + C_storage + C_network + C_gateway + C_obs + C_ops + C_risk。
  • C_gpu / C_cpu / C_mem / C_storage / C_network / C_gateway / C_obs / C_ops / C_risk：各项小时成本输入（单位：CN¥/h）。
  • m：目标毛利率（Target Margin，单位：比例 0-1，无量纲）。
  • f_pay / f_bad / f_tax：支付费率、坏账损耗、税费比例（单位：比例 0-1，无量纲）。
  • r_net = 1 - m - f_pay - f_bad - f_tax：净可用收入比例（单位：比例 0-1，无量纲）。
  • alpha（α）：缓存 token 价格相对 input 价格的折扣系数（单位：比值，无量纲）。
  • beta（β）：output token 价格相对 input 价格的溢价系数（单位：比值，无量纲）。
  • w_cache / w_in / w_out：blended 价格权重（单位：权重比例，无量纲）；系统会归一化为 hat_w_cache / hat_w_in / hat_w_out。
  • K_lat / K_rel / K_reserve：低延迟、可靠性、容量预留三项溢价系数（单位：乘数，无量纲）。
  • P_base/1M：基础每百万 token 售价（单位：CN¥/1M token），P_base/1M = Cost_1M / r_net。
  • P_final/1M：最终每百万 token 售价（单位：CN¥/1M token），P_final/1M = P_base/1M × K_lat × K_rel × K_reserve。
  • P_in / P_cache / P_out：拆分后的 input、cache、output 价格（单位：CN¥/1M token）。

二、Benchmark 与 SLA 参数（输入）

  • c：并发档位（Concurrency，单位：并发请求数）。
  • R_c：该并发下请求吞吐（单位：req/s）。
  • I_c / O_c：单请求平均 input/output token（单位：token/req）。
  • TTFT95_c / E2E95_c / ERR_c：该并发下 P95 首 token、P95 端到端、错误率（单位：s / s / 比例）。
  • T_sla / E_sla / e_sla：TTFT95、E2E95、错误率 SLA 阈值（单位：s / s / 比例）。
  • SLA_PASS_c：是否同时满足 TTFT95、E2E95、ERR 三项 SLA（单位：布尔值）。
  • S_c：可售卖比例（SLA Ratio，单位：比例 0-1）。若留空，则 PASS=1，FAIL=0；若填写则使用填写值。
  • Cache Hit：缓存命中比例输入（单位：比例 0-1；当前用于业务观察，不直接进入定价核心公式）。

三、中间量与结果指标（计算）

  • G_c = R_c × S_c：可售卖请求吞吐（Goodput，单位：req/s）。
  • T_c = 3600 × G_c × (I_c + O_c)：每小时可售卖 token 总量（Tokens/h，单位：token/h）。
  • Cost_1M,c = (C_hour / T_c) × 10^6：并发 c 下每百万 token 成本（Cost/1M，单位：CN¥/1M token）。
  • Price_1M,c：按净收入比例反推的基础售价（Price/1M，单位：CN¥/1M token）。
  • Final_1M,c：叠加 K_lat、K_rel、K_reserve 后的最终售价（Final/1M，单位：CN¥/1M token）。
  • Profit_h,c：该并发档位下每小时利润（Profit/h，单位：CN¥/h）。
  • Fail Reason：若 FAIL，展示 TTFT95/E2E95/ERR 具体超标项。

四、最优并发与决策区（计算 + 可编辑）

  • Best Concurrency：在 SLA_PASS_c = true 的档位中，优先选择 Cost_1M,c 最低的并发。
  • 并列时：先比较 G_c（更高优先），再比较 c（更低优先）。
  • Recommended Final/1M：系统根据成本与定价参数推荐的最终售价（单位：CN¥/1M token）。
  • Manual Final/1M：手动覆盖的最终售价输入（单位：CN¥/1M token）。
  • Revenue/h (Manual)：按手动售价计算的小时收入（单位：CN¥/h）。
  • Profit/h (Manual)：按手动售价计算的小时利润（单位：CN¥/h）。
  • Calculate Decision：仅重算 Pricing Decision 区，不重建 Benchmark 区。

Pricing

目标毛利率（单位：比例 0-1）。0.55 表示希望保留 55% 毛利。

支付通道费率或交易手续费（单位：比例 0-1）。

坏账、促销损耗、折扣等收入损耗（单位：比例 0-1）。

税费或其他需要从收入里扣除的比例（单位：比例 0-1）。

缓存命中价格相对 input 价格的折扣系数（单位：比值，无量纲）。越小代表 cache 越便宜。

output 价格相对 input 价格的溢价系数（单位：比值，无量纲）。越大代表输出越贵。

blended 价格里 cache token 的权重（单位：权重比例，无量纲）。

blended 价格里 input token 的权重（单位：权重比例，无量纲）。

blended 价格里 output token 的权重（单位：权重比例，无量纲）。

低延迟溢价系数（单位：乘数，无量纲）。SLA 越严格通常越高。

高可靠性溢价系数（单位：乘数，无量纲）。

容量预留溢价系数（单位：乘数，无量纲），用来覆盖冗余成本。

SLA Targets

P95 首 token 最大允许延迟（单位：秒 s）。

P95 端到端最大允许耗时（单位：秒 s）。

允许的最大错误率（单位：比例 0-1）。0.01 表示 1%。
