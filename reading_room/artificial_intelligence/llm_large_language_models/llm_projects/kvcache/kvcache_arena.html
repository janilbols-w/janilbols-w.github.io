KVCache Arena 可视化对比

对象：HiCache、LMCache、Mooncake、Pegaflow、ShadowKV。目标：按架构层级和能力边界快速选型。

1) 主层级（A/B/C/D）

A 层 算法内核层ShadowKV
B 层 框架内缓存层HiCache
C 层 引擎外 KV 服务层LMCache、Pegaflow
D 层 分布式数据平面Mooncake

A/B/C/D 表示系统定位；L0-L4 表示能力落点。

2) 次级分层（L0-L4）

L0 请求编排层
Router/Proxy、P/D 流程编排
L1 引擎连接层
vLLM/SGLang connector 适配
L2 传输执行层
NIXL/RDMA/IPC 数据通道
L3 存储与索引层
CPU/SSD/对象存储与命中管理
L4 控制与观测层
元数据、发现、监控与治理

次级分层用于细化“同属 C 层但工程重心不同”的项目差异。

2.1) 更新后的项目层次归属

项目 主层级 次级分层归属 备注
ShadowKV A L1（弱） + L3（局部） 核心仍是算法/算子层优化
HiCache B L1 + L3 框架内缓存策略与分层复用
LMCache C L1 + L2 + L3（主），L0（次） 偏能力拼装与复用路径
Pegaflow C L1 + L2 + L3（主），L0 + L4（次） 服务化和控制面更突出
Mooncake D L0 + L2 + L4（主），L3（次） 分布式传输平面与 PD 解耦核心

4) ShadowKV

优势
training-free；长上下文 decode 收益直接；可叠加 prefill 优化。
短板
平台化能力弱；跨实例共享弱；参数与任务分布敏感。

5) HiCache

优势
框架内协同自然；分层缓存直接；对对话/Agent 友好。
短板
框架绑定明显；跨节点数据平面能力有限。

6) LMCache / Pegaflow

优势
sidecar/connector 落地性强；多实例复用收益明显；可观测完善。
短板
依赖版本治理；复用率低时收益打折；RDMA 运维门槛。

7) Mooncake

优势
分布式数据平面最强；PD/xPyD 解耦完善；跨节点吞吐上限高。
短板
系统复杂度最高；网络与存储基础差时难吃满收益。

8) 选型路径（决策流）

Step 1: 单机快速验证
长上下文且 decode 压力大：先评估 ShadowKV。SGLang 体系：先评估 HiCache。
Step 2: 多实例复用升级
若前缀复用率高且要降 TTFT：评估 LMCache 或 Pegaflow。
Step 3: 跨节点与解耦上限
若瓶颈转为跨节点传输或 PD 架构：引入 Mooncake 数据平面。
Step 4: 能力叠加
推荐按 A/B -> C -> D 渐进，而不是一次性全量引入复杂系统。
