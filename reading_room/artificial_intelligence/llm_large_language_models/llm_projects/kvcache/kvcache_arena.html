KVCache Arena 可视化对比

对象：HiCache、LMCache、Mooncake、Pegaflow、ShadowKV。目标：按架构层级和能力边界快速选型。

1) 最终架构分层（L0-L4）

L0 编排与 SLO 层请求准入、TTFT/TBT 目标、Prefill/Decoding 协同
L1 Decode 关键路径层Selection / Fetch / Computation 热路径
L2 策略与索引层Attention-Based Cache Management + Hierarchical Index
L3 传输与流水层异步拉取、跨层重叠、跨节点传输
L4 存储介质层HBM / DRAM / SSD 多层协同

映射关系：A->L1，B->L2，C->L3+L4，D->L0+L3+L4。

1.1) 项目层级落位

ShadowKVA 层，主落位 L1（部分触达 L2）
HiCacheB 层，主落位 L2（部分触达 L0）
LMCache / PegaflowC 层，主落位 L3 + L4
MooncakeD 层，主落位 L0 + L3 + L4

采用“可叠加能力栈”视角，不把项目视为单一替代关系。

2) 图例（美化版）

读图顺序：先看阶段，再看关键 KV 热路径，最后看介质层级。

阶段轴：Prefill / Decoding
热路径：Selection / Fetch / Computation
策略层：Attention Cache + 索引
传输层：异步流水与跨节点调度
介质层：HBM / DRAM / SSD

2.1) 项目侧重点

ShadowKV：decode 稀疏与显存外扩 HiCache：框架内分层复用 LMCache：多实例复用与分层存储 Pegaflow：sidecar + RDMA + observability Mooncake：PD 解耦 + 分布式传输平面

4) ShadowKV

优势
training-free；长上下文 decode 收益直接；可叠加 prefill 优化。
短板
平台化能力弱；跨实例共享弱；参数与任务分布敏感。

5) HiCache

优势
框架内协同自然；分层缓存直接；对对话/Agent 友好。
短板
框架绑定明显；跨节点数据平面能力有限。

6) LMCache / Pegaflow

优势
sidecar/connector 落地性强；多实例复用收益明显；可观测完善。
短板
依赖版本治理；复用率低时收益打折；RDMA 运维门槛。

7) Mooncake

优势
分布式数据平面最强；PD/xPyD 解耦完善；跨节点吞吐上限高。
短板
系统复杂度最高；网络与存储基础差时难吃满收益。

8) 选型路径（决策流）

Step 1: 单机快速验证
长上下文且 decode 压力大：先评估 ShadowKV。SGLang 体系：先评估 HiCache。
Step 2: 多实例复用升级
若前缀复用率高且要降 TTFT：评估 LMCache 或 Pegaflow。
Step 3: 跨节点与解耦上限
若瓶颈转为跨节点传输或 PD 架构：引入 Mooncake 数据平面。
Step 4: 能力叠加
推荐按 A/B -> C -> D 渐进，而不是一次性全量引入复杂系统。
